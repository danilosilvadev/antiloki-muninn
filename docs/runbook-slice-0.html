00Decide

  • Canonical domain (blocking decision #1) — gates deploy, OG cards, secondary-domain selection, warmup. Pick it today; a decent name beats a perfect one you pick next month.
  • Name + postal line (blocking decision #2) — goes on the privacy page and, later, every cold email (CAN-SPAM requires a physical address; a virtual mailbox — e.g. a local coworking address or a mailbox service — satisfies it). The site currently says "built by Danilo"; give the configure script the rest when ready.

01Buy the domains

Three purchases (~$25–40/yr each): the canonical (the site, consented mail later — never cold), and two secondary sending domains — clearly-yours variants (think get<name>.com, try<name>.com, <name>hq.com), impersonating no one. Prefer .com for sending reputation. Two domains isolate blast radius: if one's reputation dips, the other keeps sending while it rests.

03Deploy the waitlist backend

# once: install the Supabase CLI (https://supabase.com/docs/guides/cli) and log in
supabase login

# from the antiloki-muninn repo root:
supabase link --project-ref <REF>
supabase db push                      # applies supabase/migrations (3 tables + deny-all RLS)

# secrets for the functions (generate the secret once, save it in your password manager +.env):
openssl rand -hex 32                  # → MUNINN_EDGE_SECRET
supabase secrets set MUNINN_EDGE_SECRET=<hex> ALLOWED_ORIGINS=https://<canonical-domain> LANDING_URL=https://<canonical-domain> RATE_LIMIT_PER_HOUR=20

supabase functions deploy waitlist-join unsub r
# config.toml already sets verify_jwt=false for all three (they're public endpoints);
# if your CLI version ignores it, add --no-verify-jwt to the deploy command.

06Inboxes, DNS, and warmup ON — the clock starts here

For each secondary sending domain: add it to Google Workspace, create 2–3 inboxes, then set DNS:

RecordValueNote
MXsmtp.google.com (priority 1)Workspace receiving
SPFv=spf1 include:_spf.google.com ~allTXT on the root
DKIMgenerate in Admin console → Apps → Gmail → Authenticate emailTXT, 2048-bit; turn ON after the record propagates
DMARCv=DMARC1; p=none; rua=mailto:dmarc@<canonical>TXT on _dmarc; move to p=quarantine after 2 clean weeks
  • Connect every inbox to Smartlead and switch warmup ON (start 10–20/day, ramp ~20% per step). Warmup stays on permanently, even after campaigns begin.
  • Give the canonical domain SPF/DKIM/DMARC too (it will send consented mail via Resend in slice 4) — but it never sends cold. Microsoft rejects unauthenticated bulk senders outright; Google/Yahoo enforce a 0.3% complaint ceiling. The engine's auto-pause thresholds (slice 3) assume this DNS is right.
  • First cold send is ~4 weeks from today — slice 3 will be ready long before the domains are. That's by design; don't shortcut the warmup.
