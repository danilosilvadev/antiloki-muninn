The PII concern, answered. Erasure, retention, and a budget that refuses.

The last slice flips the audit's biggest flag from "harvesting personal data" to "responsibly governed" — antiloki's own doctrine applied to its GTM. G1: "delete everything on person X" works end-to-end, leaving only a sha256: tombstone that refuses re-acquisition forever. G2: un-engaged leads and raw vendor payloads expire on a clock. G3: the budget bar becomes a circuit-breaker — enrichment/AI halt at the ceiling with an alert, not a surprise invoice. G4/D8: the secrets posture stays what it was (keys in api/.env, never in the bundle) and your data exports on demand. One function deploy, zero new keys.

how the public endpoint stays safe

Anyone can POST an email to /erasure — so it never deletes anything by itself. It acts in two layers, webhook-sink style: the edge suppresses the address instantly (worst-case abuse = unsubscribing someone, the same power the unsub link already grants) and files the request; your api turns it into a Telegram ping. You verify it's really them, then execute the deletion from the lead drawer or POST /v1/erasure. Identity checks are human, on purpose.

02The exit test — erase end-to-end · breaker trips

  1. Seed a person. Ingest a test LinkedIn URL (yours works) and let it enrich + analyze; join the waitlist with the same test email so both halves exist.
  2. Erase. Lead drawer → ⌫ erase person → confirm: erase every row. Or from the terminal: 
    curl -s -X POST 127.0.0.1:41945/v1/erasure -H 'content-type: application/json' \
  -d '{"email":"the-test@address.dev"}' | jq
#  → { "leads": 1, "waitlistMembers": 1, "rows": { ... }, "emailHashes": ["sha256:…"], … }
  3. Verify the void. The lead is gone from /leads, the member gone from /waitlist, the timeline gone with them. In the DB, only suppression rows shaped sha256:<hex> remain (reason erasure), and the cost ledger keeps its dollars with lead_id/meta nulled — money spent stays true, the person is gone.
  4. Re-ingest is refused. Send the same URL to the bot or the console → suppressed. The tombstone matched the hash; nobody had to remember why.
  5. Trip the breaker. Set MUNINN_MONTHLY_BUDGET_USD below this month's spend (Control → vendors shows it) and save → ingest a fresh URL → the lead parks with "budget ceiling reached…", Telegram gets ONE ⛔ alert, and Control shows BREAKER TRIPPED on the budget bar. Raise the ceiling back → next call flows and a ▶ reset note arrives.
  6. Exports. Settings → export & backup → four downloads (leads/waitlist × csv/json) land instantly off loopback.
