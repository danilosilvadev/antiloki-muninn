The One Door: how the auth subsystem actually holds the line
The Architect · structure lensThe Skeptic · failure lensThe Historian · git-churn lensThe Editor · merged the drafts
◆ the discovery
Every request converges on guard.ts — and nothing bypasses it. Three authors read the same 10 files through different lenses; the graph settled the argument: 0 side doors, 9 dependents, 1 uncovered branch.
1 · The shape of the doorThe Architect

Token verification and grant checking are two separate refusals on purpose: an invalid token dies before the grants table is ever touched. The dispatcher composes them in order and wraps the survivor in an audit row — the request that reaches a skill has already been refused twice.

2 · Where it would breakThe Skeptic

The uncovered branch is the audit-failure path: if the row write throws, the catch block swallows it and the mutation lands unrecorded. That is the one seam a bug can slip through — and it is exactly what the quality window flags on dispatch.ts.

3 · How it got this wayThe Historian

guard.ts changed 31 times in 6 months — every change a tightening, never a bypass. The audit wrapper arrived after a real incident, which is why it wraps everything and trusts nothing.
