muninn · slice 3 · operator runbook · 2026-07-16

The gate is built. Now the machine can send — and refuse.

This is where muninn earns its claim: every send passes SendPolicy first (suppression · consent · geo · caps · quiet hours · domain health · the kill switch), the approved ones flow into Smartlead angle campaigns on your warm secondary domains, and replies/bounces/unsubscribes come back as webhooks that pause the sequence and — for opt-outs — suppress at the edge, with your machine off. Two vendor wirings, then the exit test that proves the whole loop: approve → sent → reply pauses it < 5 min → kill switch halts everything.

prerequisite — the warmup clock

Cold sends need warm mailboxes. If you haven't run slice-0 step 6 (inboxes + DNS + Smartlead warmup ON), the code is ready but the domains aren't — start warmup now and let it finish (3–4 weeks) before real prospects get real sends. You can run this entire exit test against a test inbox you own today; only production volume waits on warmup.

01Migrate + the Smartlead key

# the slice-3 tables (refusal log, ops flags, campaign map, geo, event watermark):
supabase db push

# in the console (⚙ Settings) or api/.env — the panel is easiest:
SMARTLEAD_API_KEY   # smartlead.ai → Settings → API
# optional tuning (sensible defaults shown):
MUNINN_DAILY_SEND_CAP=30      # the conversation ceiling — new sequence pushes per day
MUNINN_QUIET_HOURS=20-8       # no pushes 20:00–08:00 on your clock
MUNINN_UTC_OFFSET=-3          # your clock's offset from UTC
MUNINN_GEO_BLOCKED=DE,CA      # cold email refused for these (UWG §7 / CASL)

Save in the panel and the runtime hot-reloads; /v1/health drops the sending line from its degraded list and the Control screen goes live.

02Wire the webhook sink (the always-on half)

# deploy the new function (config.toml already sets verify_jwt=false):
supabase functions deploy webhook-sink

# its URL, with your edge secret as the token:
https://<REF>.supabase.co/functions/v1/webhook-sink?t=<MUNINN_EDGE_SECRET>

In Smartlead → Settings → Webhooks (or per campaign), add that URL for the events: Email Sent, Email Reply, Email Bounce, Lead Unsubscribed, Spam Complaint (open/click optional). Smartlead also has per-campaign toggles for "stop on reply" and "stop on unsubscribe" — leave them ON; muninn's stop is belt-and-suspenders over them.

why this is the safety spine

The sink writes an unsubscribe / bounce / complaint straight to the suppression list the moment Smartlead posts it — no operator machine involved. Your laptop can be closed; opt-outs are still honored in under a second, and the next time muninn boots it drains the rest (reply pauses, labels, health math). Machine-off failure mode is "nothing new gets scheduled" — never "an unsubscribe didn't take."

03The exit test — approve → send → reply-pause → kill switch

Use a test inbox you own as the prospect (ingest your own alt LinkedIn, or seed a lead whose email you control). Everything below happens from the console.

#DoExpect
1Review screen → the analyzed lead is card 1 of N. Read the draft + evidence side-by-side. Press a.SendPolicy passes → "approved → campaign N"; the lead leaves the queue; Control shows the <angle> campaign and today's push count ticks up. On Smartlead, a muninn-<angle> campaign now holds the lead with its day-0/6/12 bodies as custom variables.
2Wait for Smartlead to send day-0 (or send-now in its UI for the test).the day-0 email arrives at your test inbox from a warm secondary domain — not the canonical. A sent event lands; the message row flips to sent.
3Reply to that email from the test inbox.within one scheduler tick (< 5 min, usually < 1): the sequence stops, the lead flips to replied, Telegram pings you with the reply + its label (positive/neutral/negative/ooo — a label, never an auto-action), and Smartlead stops the lead too.
4Try to approve a lead whose geo is DE, or one you've suppressed.the Review card stays and shows ⛔ SendPolicy refused: …; the refusal appears in Control's refusal log with its code. The gate is visible, not silent.
5Hit ⏸ pause all in the topbar (or Control).within one tick every campaign is PAUSED on Smartlead; a new approval is refused with pause_all; the flag is a DB row, so it survives an api restart. Hit resume → campaigns START again.
6Unsubscribe from a test send (Smartlead's link), then check Supabase.a suppressions row with reason unsub appeared from the sink; re-approving that lead is refused suppressed.
7When a day-3 LinkedIn touch comes due, muninn pings you; open the drawer, send the note in LinkedIn's real UI, click “I sent this — mark sent”.the step records as sent; muninn never automated a LinkedIn action (P1 stays yours).
slice 3 is done when

A draft you approved left a warm domain through Smartlead, a reply paused it within a tick, a refusal showed you its reason, and the kill switch stopped everything and survived a restart. That's the full sentence antiloki sells, applied to your own outreach: gated, recorded, verifiable.

Allowed sends to the UK carry a corporate-subscriber note (PECR reg 22); to careful EU geos, a legitimate-interest note — both surface in the review card. Unknown geo passes: targeting is the filter, the gate is the floor. All of this is covered by a 20+ case pure test (sendpolicy.test.ts) plus the pglite integration suite.

flag log (§ pendencies) — P5

The automated outbound sequencer is built, kept clearly on the compliant side: low volume (≤ cap/day), opt-out honored in <1s at the edge, secondary domains only, human-approve gate on every send, geo blocks, auto-pause on health.
