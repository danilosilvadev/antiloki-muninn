The loop closes. Waves go out, referrals move the line, the machine reports to you.

Slice 4 ships the last two console surfaces — the full control-center (sequence timing editor, per-angle pause, vendor spend vs budget, the compliance board) and waitlist & waves (referral leaderboard, wave manager, activation tracking) — plus the machinery under them: 3-referrals-jump-a-wave math that never fakes a position, invite codes mailed over Resend on the canonical domain, redemption tracking at the edge, thank-you v2 with WhatsApp/Telegram opt-in, and the weekly targets digest. One vendor wiring, two function deploys, then the exit test.

prerequisite — the canonical domain on Resend

Invites and the digest are consented mail and ride the canonical domain — never the secondary sending domains (two-mail-paths rule). Create a Resend account, verify the canonical domain (SPF + DKIM records Resend shows you), and pick the From: identity you'll use, e.g. Danilo <dani@yourdomain.dev>. Cold mail keeps leaving through Smartlead only.

03The exit test — wave of 10 · referral moves a position · pause-all survives restart

  1. Create the wave. Console → ≋ Waitlist → new wave: 10 slots, opens-date if you've picked one (blocking decision #5) → create wave.
  2. Preview the selection. Click the wave row — "next up" lists exactly who the math picked: referral tier first (▲), then join order. Nobody suppressed, nobody already invited.
  3. Issue. issue 10 code(s) + invites → each pick gets an 8-hex code (an invites row + invite_issued event) and a Resend email from the canonical domain with the code, the redeem link, an unsub link, and your postal line. The result line reports issued/emailed/failed honestly.
  4. Redeem. Open the invite email (use test addresses you own for the dry run) and click claim your seat → the edge function stamps redeemed_at (first click only) and lands on the manual. The wave row now shows ✓ redeemed.
  5. Activate. When a person actually starts, click mark activated on their invite row — the funnel's last column moves. A click is interest; activation is your call, deliberately.
  6. Referral moves a position. Visit /r/<someone's code>, join with three fresh emails → the thank-you page shows "banked 3 referrals — jumped 1 wave ▲", the leaderboard shows the jump, and the next wave's preview picks them first. Their join position never changed — the ranking is derived, the history honest.
  7. Pause-all survives restart. Control → ⏸ PAUSE ALL SENDING → kill the api (Ctrl-C) → npm start → Control still shows PAUSED. It's a DB flag (ops_flags), not process state.
